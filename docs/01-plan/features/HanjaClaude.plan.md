# Plan: HanjaClaude

## Overview

구몬한자 B단계 카드 학습 앱. 한자의 뜻과 음을 카드 플레이 방식으로 익히는 모바일/웹 애플리케이션.

---

## Problem Statement

한자를 학습할 때 카드 형식의 반복 학습이 효과적이다. 기존 실물 카드의 불편함(분실, 정렬, 휴대)을 해소하고, 디지털 카드로 언제 어디서나 학습할 수 있도록 한다.

---

## Goals

- 구몬한자 B단계 전체 한자를 디지털 카드로 제공
- 한자 목록에서 학습할 한자를 선택하고 순서/방향을 지정해 플레이
- 앞면(한자) / 뒷면(그림 + 뜻음) 카드 뒤집기 학습
- 앞면 또는 뒷면을 보여주고 반응을 기다린다.
- 정답 또는 오답을 지정하면 다음 한자를 진행한다.
- 직관적이고 간단한 UI로 아이도 사용 가능

---

## Scope

### 화면 구성 (4개 화면)

---
#### 1. 홈 화면 (Home Screen)

**기능**
| # | 기능 | 설명 |
|---|------|------|
| F-01 | 전체 카드 목록 표시 | 90장 전체 한자를 그리드 형태로 한 화면에 나열 |
| F-02 | 카드 선택 | 개별 카드 탭으로 선택/해제 / 전체 선택 버튼 |
| F-03 | 플레이 진입 버튼 | 1장 이상 선택 시 활성화 → 플레이 화면으로 이동 |
| F-04 | 오답 한자방 진입 버튼 | 오답 한자방 화면 이동 |
| F-05 | 옵션 진입 버튼 | 옵션 화면 이동 |

**보여지는 특성**
- 각 카드 셀: 한자(크게) + 뜻음(작게 아래) 함께 표시 — 표지 신출한자 형태
- 선택된 카드: 테두리 강조 또는 체크 표시로 시각적 구별
- 하단 고정 바: 선택 개수 표시 + 전체선택 버튼 + 플레이 시작 버튼
- 상단 우측 모서리 : 옵션 버튼
- 카드 번호(번호)가 각 셀 오른쪽 상단에 작게 표시
- 스크롤: 세로 스크롤로 전체 목록 탐색

---

#### 2. 옵션 화면 (Options Screen)

**기능**
| # | 기능 | 설명 |
|---|------|------|
| F-06 | 순서 선택 | 넘버링 순서대로 / 랜덤 순서 선택 |
| F-07 | 시작 면 선택 | 앞면(한자) 먼저 / 뒷면(뜻음) 먼저 선택 / 랜덤(한자/뜻음) 선택 |
| F-08 | 홈으로 돌아가기 | 설정 저장 후 홈 화면으로 복귀 |

**보여지는 특성**
- 선택된 카드 수 요약 표시 ("총 N장 선택됨")
- 순서 / 시작 면을 토글 버튼 또는 라디오 형태로 선택
- 하단에 크고 명확한 "홈으로" 버튼
- 단순하고 깔끔한 레이아웃 (아이도 직관적으로 조작 가능)

---

#### 3. 플레이 화면 (Play Screen)

**기능**
| # | 기능 | 설명 |
|---|------|------|
| F-09 | 앞면 표시 | 한자만 화면 중앙에 크게 표시 |
| F-10 | 뒷면 표시 | 뜻을 나타내는 그림(이미지) + 아래에 뜻음 텍스트 |
| F-11 | 카드 뒤집기 | 카드 탭(터치) → 앞면 ↔ 뒷면 전환 (flip 애니메이션) |
| F-12 | 다음 카드 | 옆으로 스와이프 → 현재 면 유지한 채 다음 카드로 이동 |
| F-13 | 진행 표시 | 현재 카드 위치 표시 (예: 3 / 10) |
| F-14 | 오답 추가 버튼 | 플레이 중 해당 카드를 오답 한자방에 추가 |
| F-15 | 오답 한자방 진입 버튼 | 오답 한자방으로 이동 |
| F-16 | 홈으로 돌아가기 | 홈 화면으로 복귀 |

**보여지는 특성**
- 카드 양면 오른쪽 상단에 카드 번호 표시
- 카드가 화면 중앙에 크게 배치 (모바일 전체 화면 활용)
- 뒤집기 시 3D flip 애니메이션으로 자연스러운 전환
- 뒷면이 시작면일때는 앞면에서 뒷면으로 이동하지 않고 바로 뒷면을 보여준다
- 상단: 진행 상황 표시 (N / 전체)
- 하단 또는 카드 모서리: 오답 추가 버튼 (별 모양 또는 "+" 아이콘)
- 배경은 단색 또는 연한 패턴으로 카드에 집중되도록

---

#### 4. 오답 한자방 (Wrong Answer Room)

**기능**
| # | 기능 | 설명 |
|---|------|------|
| F-17 | 오답 카드 목록 | 추가된 오답 한자를 그리드로 표시 |
| F-18 | 카드 삭제 | 카드 길게 누르기 → 삭제 확인 후 제거 |
| F-19 | 오답 카드 플레이 | 오답 카드만 선택해 플레이 화면(F-08~F-14)으로 진입 |

**보여지는 특성**
- 홈 화면과 동일한 그리드 레이아웃 (한자 + 뜻음 표시)
- 빈 상태일 때: "아직 오답 한자가 없어요" 안내 메시지 표시
- 길게 누르면 삭제 모드로 전환 (X 표시 등장)
- 상단에 "전체 플레이" 버튼으로 모아진 카드 즉시 플레이 가능

---

## User Stories

- 학부모/아이로서, 홈 화면에서 오늘 공부할 한자를 선택하고 카드 학습을 시작하고 싶다.
- 학습자로서, 한자(앞면)를 보고 뜻과 음을 마음속으로 생각한다. 터치해서 정답을 확인한다.
- 학습자로서, 뜻과 음(뒷면)을 보고 한자를 마음속으로 생각한다. 터치해서 정답을 확인한다.
- 학습자로서, 여러 장을 빠르게 훑어볼 때 스와이프로 다음 카드로 넘기고 싶다.
- 학습자로서, 랜덤 순서로 섞어 학습 효과를 높이고 싶다.

## Data Source

### hanjab.csv

- **경로**: `data/hanjab.csv`
- **총 카드 수**: 90장 (구몬한자 B단계)
- **컬럼 구조**:

| 컬럼 | 설명 | 예시 |
|------|------|------|
| 번호 | 카드 번호 (1~90) | `1` |
| 한자 | 한자 문자 | `目` |
| 뜻 | 한자의 뜻 (한국어) | `눈` |
| 음 | 한자의 음 (한국어) | `목` |

- **활용 방법**: 빌드 시 CSV를 파싱하여 JSON으로 변환하거나, 런타임에 직접 fetch하여 사용
- **이미지**: CSV에 그림 컬럼 없음 → 뒷면 그림은 별도 에셋 디렉토리(`public/images/hanja/`)에 `{번호}.png` 형식으로 관리하거나, 초기 버전은 뜻 텍스트만으로 대체 가능

### 카드 데이터 모델 (앱 내부)

```json
{
  "id": 1,
  "hanja": "目",
  "meaning": "눈",
  "pronunciation": "목",
  "imagePath": "/images/hanja/1.png"
}
```

## Technical Approach

- **타입**: 모바일 우선 웹 앱 (PWA) 또는 React Native
  - 추천: Next.js (PWA) - 별도 앱 설치 없이 브라우저에서 바로 사용 가능
- **한자 데이터**: `data/hanjab.csv` 활용 (90장, 번호/한자/뜻/음). 빌드 시 JSON 변환 또는 런타임 fetch
- **카드 UI**: CSS flip animation 또는 React Spring/Framer Motion
- **스와이프**: Touch event 또는 Swiper.js
